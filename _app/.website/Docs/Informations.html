    Sobre & Ferramentas

    Bem, a parte de Criação de Websites é a parte que eu gosto e uma das mais “simples” entre aspas em fim, a criação de um Website é necesario aprender Sobre JavaScript, Html, Php e etc Caso queira entrar nesse universo de Websites é necesario saber essas linguagems. Se você prefere mais Html e JavaScript não é necesario saber Php ou ao contrario.

    Mas uma coisa legal de Websites é que existe MUITAS ferramentas de desenvolvemento de Websites como exemplo Jekyll, Jekyll é uma ferramenta utilizada para criação de sites simples usando Ruby como linguagem de base, mas não existe só mente Jekyll Existe Ruby On Rails que também é um metodo de criação de site mais complexa

    Hospedagem

    O sistema de Hospedagem é utilizado como o nome fala Hospedar a aplicação existe também muitos lugares para fazer isso como uma delas o proprio Github com o Github Pages Tem como você criar um site pelo Github usando Github Pages ou Hospedar um Site neles também existe outros metodos pagos e de graça para hospedar o seu Site como VPS (Servidor Virtual Privado), Cloud (Hospedagem na nuvem), GoDaddy, Umbler, Heroku e muito mais (NOTA: algumas de graça ou pagas).

    Dominio & Dominio Proprio

    Bem como o mundo não é um mar de rosas os lugares que hospedam os Websites Precisão de um Proprio dominio como esse: https://gabrielramires.github.io/MinecraftServerMenu isso é um dominio NÃO PROPRIO sendo assim um dominio que ira aparecer seu nome e o lugar github.io. Mas existe um metodo de conseguir mudar esse dominio comprando um dominio exemplos: .com, .org, .co e etc.

    Usando CMD ou Power Shell

    1. Intalando Jekyll em Ruby

      • gem install Jekyll


    2. Construindo um Projeto em Jekyll usando o Eazy_Jekyll!

      • git clone "CoolWebsite"

    3. Abrindo o Projeto no CMD ou Power Shell

      • cd CoolWebsite


    4. Executando um Servidor Local com Jekyll

      • Execuções:

        1. Normal:

          • jekyll server


        2. Com alguma informação a mais:

          • jekyll server --port=4001


            (por exemplo baseurl caso você tenha o projeto no Github você tera que alterar o baseurl para não gerar erros)
